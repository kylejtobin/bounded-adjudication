# Intentional Discovery

Top-down discovery for projects where structural information lives in the author's stated commitments rather than in the artifact's observable structure. Creative writing, editorial content, design systems, brand guidelines, documentation standards, policy documents. The artifact shows patterns but cannot tell you which are intentional and which are accidents.

This reference teaches what to listen for in the user's stated commitments and how to distinguish commitment from habit. Elicit commitments before reading the work. The gap between stated commitment and observed practice is where gates live.

## Elicit Commitments Before Reading

Ask the user what rules they have set for themselves before examining any work product. What POV? What tense? What voice? What conventions are they honoring? Which are they deliberately breaking? What does the style guide say? What does the brand spec require?

Stated rules are invariant candidates. The user said "always" or "never." Test each one: is this truly always/never, or is it usually? A stated rule with legitimate exceptions is a gate, not an invariant.

## Separate Intentional Patterns from Accidents

After the user states their commitments, read the work and identify observable patterns. Present them to the user. "I see consistent use of short declarative sentences. Is that intentional?" The answer sorts patterns into commitments and habits.

Intentional patterns confirmed by the user become evidence for gates: allowed shapes when present, disallowed shapes when violated. Accidental patterns the user didn't know about are opportunities: does the user want to formalize them? If yes, they become evidence shapes. If no, they are noise and should not appear in the rubric.

## The Gap Is Where Gates Live

When the user's stated commitment differs from what the work actually does, that gap reveals where judgment is needed. The user says "I write in close third person limited." The manuscript has two scenes with omniscient perspective. Ask: are those intentional departures or mistakes?

If intentional: the gate needs an approved mechanism for deliberate departures. What makes a departure legitimate? The user's answer becomes the mechanism.

If mistakes: the gate's disallowed evidence includes the specific shape of the violation. What does omniscient intrusion look like in close third? Another character's internal state without attribution. That's an evidence shape.

## Convention Is Default Authority

Every domain has conventions. Genre conventions in fiction. Style guide conventions in documentation. Design system conventions in UI work. Brand guidelines in marketing content. These are the default authority topology until the user explicitly departs from them.

Ask: what conventions apply to this work? Which are you following? Which are you deliberately breaking? Followed conventions are part of the authority topology. Deliberately broken conventions need approved mechanisms explaining why the departure is sanctioned. Unknowingly broken conventions are potential disallowed evidence shapes.

## Stated Rules Are Invariant Candidates

When the user says "I never" or "we always," that's an invariant candidate. Test it the same way as structural invariants: can you name a legitimate case where this shape is correct (or incorrect)? If the user hesitates, it's probably a gate with evidence shapes rather than a true invariant.

## Unstated but Consistent Patterns Are Candidate Gates

Patterns the user didn't state but consistently follows are candidate gates needing confirmation. "Every chapter opens with a sensory detail. You didn't mention that as a rule. Is it one?" If the user says yes, it becomes a gate with evidence shapes. If the user says no, it doesn't enter the rubric regardless of how consistent the pattern is.

Do not formalize patterns the user doesn't recognize as commitments. The rubric reflects the user's declared standards, not the agent's observations about their habits.
